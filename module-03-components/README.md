# Module 3 · Components

> **You'll learn:** how to split a UI into self-contained pieces and wire them together - props to pass data down, emits to send events up, slots to hand markup in. This is the architecture of every Vue app you'll ever read.

**Builds on:** [Module 2 · Reactivity](../module-02-reactivity/)

## Lessons

1. [Your First Component](./01-your-first-component.md) - extracting, reusing, and a feel for where to cut *(~15 min + exercise)*
2. [Props](./02-props.md) - passing data down, and why the street is one-way *(~15 min + exercise)*
3. [Emits](./03-emits.md) - sending events up, completing the loop *(~15 min + exercise)*
4. [Slots](./04-slots.md) - components with holes the caller fills *(~15 min + exercise)*

Take them in order - each lesson deliberately runs into a wall that the next one knocks down, and the exercises rebuild the Snack Cart into proper components piece by piece.

## When you're done

Your Snack Cart is a component tree - reusable rows driven by data, a parent that owns all the state, and presentational cards that wrap anything. Adding a product means editing an array, not markup. After this module the course path **branches**: modules 4 (Forms), 5 (Routing), 6 (State), and 7 (APIs) are open in any order.
