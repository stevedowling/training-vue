# Module 5 · Routing

> **You'll learn:** how one Vue app becomes a many-page *site* - Vue Router setup, pages with real URLs, dynamic routes for detail pages, navigation from code, and a 404 that catches everything else.

**Builds on:** [Module 3 · Components](../module-03-components/) · Independent of modules 4, 6, 7, take in any order.

## Lessons

1. [Pages and links](./01-pages-and-links.md) - install the router, turn components into pages, navigate without reloading *(~15 min + exercise)*
2. [Dynamic routes](./02-dynamic-routes.md) - `/products/:id`, the list → detail pattern, and the component-reuse trap *(~15 min + exercise)*
3. [Navigation in code + 404s](./03-navigation-in-code-and-404s.md) - `router.push` vs `replace`, and owning your broken URLs *(~15 min + exercise)*

Take them in order - the exercises turn the sandbox into a real site: shop page, product detail pages with next/previous, a checkout flow, and a 404 with personality.

## When you're done

Every part of your app has an address - bookmarkable products, a shareable shop, sensible back-button behaviour, and no URL that dead-ends in a blank page. You'll also have met the one thing routing *can't* do well (carrying state between pages), which is Module 6's cue.
